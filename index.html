Git Command

git config --global user.email "souvik.sarkar.cs@gmail.com" --> To add email

git config --global user.name "Souvik-025" --> To add username

git config --global --list --> to List all global information

git config --list --> To list all envirenmental data

git clone --> To clone a repository in our local machine

git status --> Use check status of our git repository

git add filename --> to add new or changed file in our working directory to the git staging area.

git add . --> to add all files present in the repository

git commit -m "some message" --> to commit changes of our directory to the git repository

git push origin main --> To upload local repository content to remote repository(git repository)

git push -u origin main -->(u :- used for upstream) Used to set origin main name for future use so that everytime we are pushing we don't have to provide the name

init :- used to create a new git repo

git init --> Used to create a new git repo in our local repo

git remote add origin url/link --> use to add a new remote with the name origin(convetion to keep name as origin) at the link

git remote -v --> Used to verify the remote or in other word to check what is present in the remote

git branch --> Used to check in which branch we are present

git branch -M mymain --> Used to change the branch name to mymain

git checkout -b --> Used to create new branch

git checkout --> Used to switch between branches

git branch -d --> Used to delete branch but if the present branch and deleting branch is same then it will not work

git push origin --> Used to push repo to that particular branch in github

git diff --> Used to check difference between the current branch and the given branch

git pull origin --> Used to pull from github to local repo of that particular branch

git merge main --> Used to merge current branch with the main branch with all the changes

git reset filename --> Used to revoke the changes in that particular file after add

git reset --> Used to revoke all the changes in our repo after add

git reset HEAD~1 --> Used to revoke all the changes in repo after commit

git log --> Used to get the log of all the changes along with the hashcode

git reset --> Used to get back to that particular change state

git reset --hard --> Used to revoke all the changes from editor terminal and github and get back to normal with effect in code
